# Idea 10: RFC Impact Simulator

Before writing code, an engineer posts an RFC/design doc to Slack. The agent reads it, extracts the proposed interface changes, then uses Sourcegraph to simulate the impact: 'If you add parameter X to FooService.bar(), here are the 47 callers across 12 repos that would need updating. Here are 3 callers that pass bar() as a callback — they'll break silently. Here's a similar migration that happened 6 months ago (found via commit_search) — it took 3 weeks.' The agent provides a reality check on design proposals BEFORE implementation starts, using the actual codebase as ground truth.
