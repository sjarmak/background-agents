# Idea 19: Codebase Socratic Tutor

Instead of answering questions, the agent asks them. When a junior dev opens a PR, the agent uses Sourcegraph deepsearch to understand the broader context, then posts thought-provoking questions: 'I see you added a cache here. Did you notice that PaymentService (repo-Y) has a similar cache at line 42 that was removed in commit abc because of stale data issues? What makes your case different?' The structural shape: the agent's output is questions, not answers. It teaches by connecting the developer's work to organizational lessons embedded in the code history.
