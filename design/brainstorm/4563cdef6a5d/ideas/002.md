# Idea 2: Living Architecture Narrator

A scheduled agent that uses Sourcegraph deepsearch weekly to build and maintain a living narrative of how the system architecture actually works — not documentation that drifts, but a continuously regenerated 'state of the architecture' that diffs against last week's version. Uses commit_search and diff_search to explain WHY things changed. Posts architectural drift reports to a Slack channel: 'This week: auth service gained 3 new callers, payment module's interface narrowed, the deprecated UserV1 type gained 2 new references (regression).' Sourcegraph's temporal code intelligence (diffs, commits, revisions) is the unique enabler.
