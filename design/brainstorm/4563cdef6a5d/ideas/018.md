# Idea 18: Reverse Dependency Canary

Inverted trigger model: when an UPSTREAM library releases a new version (detected via GitHub release event), the agent uses Sourcegraph to find every downstream repo that imports it, then for each: checks if the new version's changelog mentions breaking changes, reads the actual diff via compare_revisions, and tests whether each downstream repo's usage patterns match the changed APIs. Pre-emptively opens PRs in downstream repos to bump the version and fix breakage BEFORE anyone notices. The shape: upstream event triggers downstream multi-repo action, powered by cross-repo dependency tracing.
