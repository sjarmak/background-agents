# Idea 27: Team Boundary Erosion Monitor

Uses Sourcegraph to track cross-team code dependencies over time. Maps which team's repos import from which other team's repos (via find_references + list_repos + contributor info). Detects when team boundaries are eroding: 'Team Alpha's services now have 47 direct imports from Team Beta's internal packages (up from 12 last quarter). These 5 symbols are becoming de facto shared APIs without being designed as such.' Posts quarterly reports to engineering leadership. Shape: organizational/sociological analysis derived from code dependency patterns — uses Sourcegraph as a sociotechnical system sensor.
