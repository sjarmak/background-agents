# Idea 4: Tribal Knowledge Excavator

When a Slack thread includes 'does anyone know how X works?' or 'who built the Y system?', the agent activates. Uses Sourcegraph's get_contributor_repos + commit_search + go_to_definition to trace the authorship and evolution of the code in question. Reconstructs the 'tribal knowledge' — who originally designed it, what decisions were made (from commit messages and diffs), how the implementation drifted from original intent. Replies in-thread with a narrative history, not just a link to a file. The unique value: Sourcegraph's temporal + authorship intelligence excavates institutional memory that lives nowhere else.
