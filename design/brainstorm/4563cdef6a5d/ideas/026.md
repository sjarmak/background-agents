# Idea 26: Postmortem Evidence Compiler

After an incident is resolved, someone writes 'postmortem' in the incident Slack channel. Agent uses Sourcegraph to automatically compile the evidence package: (1) all code changes in the affected services during the incident window (diff_search + compare_revisions), (2) the full call chain from entry point to failure (go_to_definition + find_references), (3) similar past incidents found via commit_search for related fix patterns, (4) a timeline of code changes leading up to the incident. Outputs a structured postmortem template pre-filled with code evidence. Shape: the agent compiles forensic evidence after resolution, not during — it's a historian, not a responder.
