# Idea 1: Dependency Impact Oracle

When a library team pushes a breaking change, the agent automatically uses Sourcegraph find_references across ALL downstream repos to build a complete blast radius map — every caller, every test, every integration point. Posts a cross-repo impact report to Slack with affected team owners, suggested migration paths, and estimated effort. The trigger is a git push to a shared library, not a human request. Sourcegraph's cross-repo references are the only way to do this without cloning hundreds of repos.
