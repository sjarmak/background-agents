# Idea 13: Codebase Question Routing via Expertise Graph

Agent builds and maintains a graph of 'who knows what' by mining Sourcegraph contributor data + commit history + code ownership patterns. When someone asks a question in Slack, instead of (or before) answering directly, it routes to the right human: 'Based on commit history, @alice is the primary author of this subsystem and @bob recently refactored it. CC'ing them.' The agent's role is connector/router, not answerer. Uses get_contributor_repos + commit_search to build the expertise topology. Fundamentally different shape: the agent makes humans more effective at helping each other rather than replacing them.
