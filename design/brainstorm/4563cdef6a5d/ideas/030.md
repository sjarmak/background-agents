# Idea 30: Knowledge Graph Materializer: Code as Queryable Database

Agent uses Sourcegraph's full tool suite to materialize the codebase into a queryable knowledge graph (stored as a lightweight DB or JSON). Nodes are symbols (functions, types, services), edges are relationships (calls, imports, implements, extends). Attributes include: last modified date, test coverage, number of callers, contributor count. Slack becomes a natural language query interface: 'Which functions have more than 10 callers but no tests?' 'What's the longest dependency chain from the API gateway to the database?' The agent translates NL to graph queries. Shape: the codebase becomes a structured database that non-technical queries can explore. Sourcegraph provides the raw data; the agent materializes and indexes it.
