# Idea 11: Convention Drift Detector

Agent runs on a schedule, using Sourcegraph nls_search + keyword_search to detect when different repos solve the same problem in divergent ways. Example: 'Repos A,B,C use structured logging with Zap. Repos D,E switched to slog last month. Repos F,G still use fmt.Printf.' Clusters patterns by semantic similarity, identifies the emerging consensus vs. the outliers, and posts 'convention drift reports' to Slack with a recommendation to standardize. Unlike a linter (which enforces known rules), this DISCOVERS implicit conventions by mining actual cross-repo usage patterns.
