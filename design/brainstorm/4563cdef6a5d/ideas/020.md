# Idea 20: Architecture Fitness Function Runner

Agent implements continuous architecture fitness functions as described by evolutionary architecture. Scheduled weekly, it uses Sourcegraph to measure structural metrics across the org: dependency direction violations (service A shouldn't import from service B), layering violations (data layer calling presentation layer), circular dependency chains across repos, interface complexity growth. Each metric is a 'fitness function' with a threshold. Posts a dashboard to Slack showing which architectural invariants are holding vs. degrading. The shape: the agent is a continuous architectural test suite, not a one-off analyzer.
