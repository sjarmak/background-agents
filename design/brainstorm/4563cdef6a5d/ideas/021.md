# Idea 21: Semantic Changelog Generator from Cross-Repo Diffs

Triggered by a release tag. Agent uses Sourcegraph diff_search and compare_revisions to gather all changes since last release across ALL repos that contribute to the product. Then uses deepsearch to understand the user-facing impact of each change. Generates a semantic changelog: not 'updated foo.ts line 42' but 'Users can now export reports as CSV (implemented across api-service and frontend-app in 4 PRs).' Groups changes by user impact, not by repo or commit. The shape: cross-repo diff aggregation → user-facing narrative synthesis.
