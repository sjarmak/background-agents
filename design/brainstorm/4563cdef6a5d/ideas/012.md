# Idea 12: Context-Aware PR Reviewer with Org Memory

Goes beyond standard AI code review by using Sourcegraph to inject organizational context. When reviewing a PR, the agent searches for: (1) how similar changes were done elsewhere (keyword_search + nls_search), (2) whether the same bug was fixed before in another repo (commit_search), (3) whether the changed API has known gotchas based on reference patterns (find_references). The review comment includes 'This pattern was tried in repo-X and reverted in commit abc123 because...' — the agent has institutional memory via Sourcegraph's history, not a vector DB.
