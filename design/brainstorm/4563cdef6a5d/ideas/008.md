# Idea 8: Migration Wavefront Planner

For large-scale migrations (e.g., switching auth libraries), agent uses Sourcegraph to build a dependency-ordered migration plan across repos. Maps the call graph via go_to_definition/find_references, identifies leaf repos (no downstream dependents) as safe to migrate first, then works inward. Generates a multi-week wavefront schedule: 'Week 1: repos A,B,C (leaf). Week 2: repos D,E (depend only on already-migrated). Week 3: ...' Each week's batch is a GitHub Actions workflow that creates PRs in parallel. The agent IS the migration coordinator, not a tool within one.
