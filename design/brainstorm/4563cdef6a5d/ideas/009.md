# Idea 9: Incident Root Cause Tracer via Code Archaeology

When an incident is declared in Slack, the agent doesn't just search logs — it uses Sourcegraph diff_search and commit_search to find all code changes in the last 48 hours that touched the failing service's call path. Uses find_references to trace the full request path from entry to failure point. Cross-references with contributor info to identify who changed what. Posts a 'suspect change list' ranked by proximity to the failure, with direct links to the diffs. This is code-level incident forensics, not log analysis — using Sourcegraph's temporal diff intelligence.
