# Idea 14: Speculative Pre-Review: Agent Reviews Code Before the Developer Finishes

Triggered by a WIP commit push (draft PR or branch push). Agent doesn't wait for a review request — it immediately uses Sourcegraph to understand the change in context: checks cross-repo callers, identifies missed test cases from similar implementations elsewhere, spots patterns that were reverted in other repos. Posts early feedback as a Slack DM to the developer (not on the PR — it's not ready yet). The structural novelty: feedback arrives DURING development, not after. The trigger is git push, the output is private/ephemeral, the goal is course-correction not review.
