# Idea 25: Cross-Repo Invariant Verifier

Engineers declare cross-repo invariants in a config file: 'Every service that imports auth-lib MUST call auth.init() in its main function', 'No repo should import both old-db-client and new-db-client', 'Every gRPC service must have a health check endpoint'. Agent runs on every PR and on a schedule, using Sourcegraph keyword_search + find_references to verify these invariants hold across the entire org. Violations are posted to PR comments (if triggered by a PR) or Slack (if scheduled). Shape: user-defined cross-repo assertions enforced by continuous Sourcegraph queries — like linting rules but for organizational-level architectural constraints.
